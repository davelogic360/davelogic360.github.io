This section contains a few useful notes and commands for navigating around Git:

Creating new projects

  • In Visual Studio Code> File > Open Folder
  • Navigate to ..Documents > Projects and create a new folder for the project
  • Open Folder
Command Description
<git init> Initialises a Git repository
<git clone https://github.com/davelogic360/codecademy.git> Create a local copy of the repository

Basic Commands

Command Description
<git status> Status check
<git add [file-name.txt]> Adds a file ready for staging
<git add -A> All file changes added to staging area
<git commit -m "[Commit message]"> Commits changes and add a message
<git rm -r [file-name.txt]> Removes a file or folder

Branching and Merging

Command Description
<git branch> List all branches (asterisk is current)
<git branch -a> List all local and remote branches
<git branch [branch name]> Create a new branch
<git branch -d"> Deletes a branch
<git branch origin --delete> Delete a remote branch
<git branch -m [old branch name] [new branch name]> Rename a local branch
<git checkout -b [branch name]> Create a new branch and switches to it
<git checkout -b [branch name] origin/[branch name]> Clone a remote branch and switch to it
<git checkout [branch name]> Switch to a branch
<git checkout -> Switch to the branch last checked out
<git checkout --[file-name.txt]> Discard changes to a file
<git merge [branch name]> Merge a branch to the active branch
<git merge [source branch] [target branch]> Merge a branch to a target branch
<git stash> Stash changes to a dirty working directory
<git stash clear> Remove all stashed entries

Updating Projects

Command Description
<git push origin [branch name]> Push a branch to your remote repository
<git push -u origin [branch name]> Push changes to remote repository and remember branch
<git push> Push all change to remote repository
<git push origin --delete [branch name]> Deletes a remote branch
<git pull> Pull change from remote repository
<git pull origin [branch name]> Pull changes from remote respository for a given branch

To merge a branch back in to main:

  1. git checkout main - Switch to main
  2. git pull origin main - Get latest changes from remote
  3. git merge [branch name] - merge change from branch to main
  4. git push origin main - push changes back to remote

Git Tools

Command Description
<git log> View changes
<git log --summary> View detailed changes
<git log --oneline> View brief changes
<git diff [source branch] [target branch]> View changes before merging

HTML Hierachy

W3Schools HTML Tags Reference: Click here

Tag Name Description
<html> HTML Container for all the HTML code on the page
<head> Head Container for metadata such as Titles, links to CSS and metadata
<title> Head Title Used to name the page in browser
<link> Head Link Used to link page to things like style sheet and fonts
<body> Body Incs headings, paragraphs, images, hyperlinks, tables, lists, etc
<header> Body Header Often contains heading, logos and navigation
<nav> Header Nav Often used for the main site navigation
<main> Body Main Page content unique to that page
<section> Main Section Defines a section of infromation on the page
<articles> Section Articles Information relevant to the section its in
<aside> Articles Aside The aside content should be indirectly related to the surrounding content.
<footer> Body Footer Typically contains contact and author information

Table Tags

Tag Name Description
<table> Table The wrapper element for all HTML tables.
<thead> Table Head The set of rows defining the column headers in a table.
<tbody> Table Body The set of rows containing actual table data.
<tr> Table Row The table row container.
<td> Table Data The table row container.
<tfoot> Table Foot The set of rows defining the footer in a table.

Table Tags

Tag Name Description
<colspan> Column Span Defines how many columns a td element should span.
<rowspan> Row span Defines how many rows a td element should span.
